# Skill 评测技能

## 语言约束（全局）

**所有面向用户的输出必须使用中文**，包括但不限于：提示语、选项表、确认问句、错误信息、结果摘要。禁止输出英文句子或中英混排段落（代码片段、模型名称、URL 等专有名词除外）。

**禁止暴露执行过程中的思考过程**。只能输出以下允许的规范化内容：

| 允许的输出类型 | 示例 |
|----------------|------|
| 阶段进度表 | TodoWrite 更新 4 阶段进度 |
| 选项表 | 场景选择、模型选择、评测集来源选择等 Markdown 表格 |
| 单行提问 | "请提供 Skill 名称"、"请提供自定义模型的 API URL" |
| 确认问句 | "根据你的描述，你想要「快速测试单个 Skill」，是这样吗？" |
| 登录提示 | "请在浏览器完成授权，然后输入授权码：" + 登录链接 |
| 结果摘要 | 任务摘要表、评测报告链接 |

**禁止输出**：

- 思考过程说明（"用户说的是 xxx，我需要先 xxx"、"这里有两个分支，我选择 xxx"）
- 执行状态描述（"正在检查 Token"、"现在读取 xxx 文档"、"已加载历史任务"）
- 推理结论回放（"从输入提取出 Skill 名称为 xxx"、"场景确认为 xxx"）
- 流程跳转说明（"进入场景识别阶段"、"跳转到阶段2"）
- 任何"我接下来要做什么 / 我刚刚做了什么"的句子

**判断标准**：这句话是在直接向用户提问、请用户选择、或展示最终结果吗？不是 → 不输出。

## 🔇 加载即静默（最高优先级）

**SKILL 加载后，第一件事是调用 TodoWrite 更新阶段1进度，然后静默执行鉴权检查**。

在调用 TodoWrite 之前，禁止输出任何文本，包括但不限于：

- "用户已描述具体需求"、"按 xxx 分支处理"
- "先静默执行 Token 检查"
- "正在加载历史任务"
- "用户已描述具体需求，跳过历史任务展示"
- "已识别 Skill 名称为 xxx"
- "接下来识别意图"
- "Token 检查通过 / 鉴权完成 / 未发现历史任务"
- "现在读取 xxx 文档"
- "进入场景识别 / 构建配置阶段"
- 任何说明"我接下来要做什么 / 我刚刚做了什么"的句子

**首个输出必须是阶段1进度表**（调用 TodoWrite 工具）：

```
TodoWrite([
  { content: "确认评测场景", status: "in_progress", activeForm: "检查登录状态" },
  { content: "配置评测对象", status: "pending", activeForm: "配置评测对象" },
  { content: "准备评测数据", status: "pending", activeForm: "准备评测数据" },
  { content: "提交并查看报告", status: "pending", activeForm: "提交并查看报告" }
])
```

> **注意**：初始调用时，阶段1 的 `activeForm` 用该阶段第一个步骤的描述（"检查登录状态"），后续各阶段的 `activeForm` 用阶段名称本身。执行到具体步骤时，再按对照表更新为对应步骤的 `activeForm`。

**各步骤执行时更新 activeForm**：

| 阶段 | 步骤 | activeForm |
|------|------|------------|
| 阶段1 | 检查 Token | 检查登录状态 |
| 阶段1 | 触发登录 | 等待浏览器授权 |
| 阶段1 | 查询历史任务 | 查询历史任务 |
| 阶段1 | 识别评测场景 | 分析评测需求 |
| 阶段2 | 选择 Skill 获取方式 | 确认 Skill 来源 |
| 阶段2 | 搜索/识别 Skill | 确认 Skill 中 |
| 阶段2 | 打包上传 Skill | 上传 Skill 包 |
| 阶段2 | 选择运行框架 | 选择运行框架 |
| 阶段2 | 选择驱动模型 | 选择驱动模型 |
| 阶段2 | 平台推荐评委模型 | 确认评委模型 |
| 阶段2 | 选择其他评委模型 | 选择评委模型 |
| 阶段3 | 选择评测集来源 | 准备评测数据 |
| 阶段3 | 上传评测集 | 上传评测数据 |
| 阶段3 | 合成评测集 | 生成评测数据 |
| 阶段3 | 预览评测集 | 确认评测数据 |
| 阶段4 | 展示任务摘要 | 确认评测配置 |
| 阶段4 | 提交评测任务 | 提交评测任务 |
| 阶段4 | 等待评测结果 | 等待评测完成 |
| 阶段4 | 展示评测报告 | 查看评测报告 |

**进度表之后的第二个输出**根据触发条件选择：

| 条件 | 第二个输出 |
|------|---------|
| 用户已描述具体需求 → 新建评测 | 场景推断确认（"根据你的描述，你想要「xxx」，是这样吗？" + 选项表）|
| 用户未描述具体需求 + 无历史任务 | 自然语言引导（"请描述你的评测需求，例如：…"）|
| 用户未描述具体需求 + 有历史任务 | 历史任务表 |
| Token 失效需手动授权 | 登录链接 |

**判断方法**：写第二句文本前，先自问"这句话是在直接向用户提问/请用户选择吗？"——不是则不写。

**首个输出格式硬性要求**：

| 要求 | 说明 |
|------|------|
| 任何选项必须用 Markdown 表格 | 禁止用 `1. xxx\n2. xxx` 列表、禁止用 `• xxx` 项目符号 |
| 阶段1 只问场景，不问其他 | 不要在场景选择里掺杂"是否已有评测集"、"用什么模型"等阶段2/3 的问题 |
| 场景模糊时走 5 场景选择表 | 5 选项必须与 [scene-detection.md 步骤3 分支 B](./processes/scene-detection.md) 完全一致：1 快速测试单个 Skill / 2 对比多个同类 Skill / 3 同一个 Skill 不同模型驱动的表现 / 4 同一个 Skill 不同运行框架的表现 / 5 不确定（希望得到引导）|
| 禁止"路由说明" | 不要打印"已路由到 skill-evaluation"、"因为你的目标是 xxx" 这类元解释 |

## 概述

协助用户对本地 Skill 进行自动化评测，支持单 Skill 验证、多 Skill 横评、多模型横评、运行框架横评、版本对比等场景。

**核心原则**：自动识别优先，渐进式确认，**实现细节与内部流转对用户不可见**。

**适用场景**：
- 评测某个 Skill 的执行效果
- 比较不同模型驱动同一 Skill 的效果、速度、Token 消耗
- 比较多个同类 Skill 或同一 Skill 多版本的差异
- 比较同一 Skill 在不同运行框架下的表现

**不适用场景**（引导用户改用其他方式）：
- 代码评审 → 建议使用代码审查工具
- 系统性能压测 → 建议使用性能测试框架
- 模型训练/微调 → 不属于评测范畴

## 流程感知

**目标**：让用户清晰了解评测流程的全貌和当前进度。

### 入口流程

**进入本 skill 后第一件事**：

1. **调用 TodoWrite 打印阶段1进度表**（无论用户输入是什么）
2. **静默执行鉴权检查**
3. 根据触发条件展示对应的首个输出（历史任务表 / 场景选择表 / 自然语言引导）

**阶段1进度表**（入口处调用 TodoWrite）：

```
TodoWrite([
  { content: "确认评测场景", status: "in_progress", activeForm: "检查登录状态" },
  { content: "配置评测对象", status: "pending", activeForm: "配置评测对象" },
  { content: "准备评测数据", status: "pending", activeForm: "准备评测数据" },
  { content: "提交并查看报告", status: "pending", activeForm: "提交并查看报告" }
])
```

**步骤执行时动态更新 activeForm**：每进入一个新步骤，调用 TodoWrite 更新当前阶段的 activeForm 为该步骤描述。

### 阶段进度跟踪

**进入每个新阶段时调用 TodoWrite 更新进度表**：

1. **阶段1**：入口处立即调用（in_progress 状态）
2. **阶段2**：场景确认完成后调用（阶段1 completed，阶段2 in_progress）
3. **阶段3**：评测对象配置完成后调用
4. **阶段4**：评测数据准备完成后调用

**步骤级进度更新**：每执行到一个新步骤时，调用 TodoWrite 更新当前阶段的 activeForm 为该步骤描述。

例如，阶段2 执行到「打包上传 Skill」步骤时：

```
TodoWrite([
  { content: "确认评测场景", status: "completed", activeForm: "检查登录状态" },
  { content: "配置评测对象", status: "in_progress", activeForm: "上传 Skill 包" },
  { content: "准备评测数据", status: "pending", activeForm: "准备评测数据" },
  { content: "提交并查看报告", status: "pending", activeForm: "提交并查看报告" }
])
```

### 进入步骤时

**⚠️ 阶段执行规则**：

1. **默认按序执行**：从初始化阶段开始，按顺序推进
2. **用户指明可跳转**：用户明确要求进入某阶段或某任务时可直接跳转
3. **阶段内自动流转**：当前阶段任务全部完成后自动进入下一阶段

**⚠️ 工具使用限制**：

| 工具 | 是否允许 | 说明 |
|------|----------|------|
| `AskUserQuestion` | **禁用** | 本 skill 所有交互均通过直接输出文本 + Markdown 表格完成，不得使用 AskUserQuestion 工具 |
| `TodoWrite` | 允许 | 用于更新阶段进度 |
| `Bash` | 允许 | 用于执行脚本命令 |
| `Read` / `Edit` / `Write` | 允许 | 用于文件操作 |

## 交互规范

### 输出时机原则

**只展示核心步骤，询问用户的问题**。其他一切（推理、流转、状态、结果）保持静默或合并到问题中。

| 时机 | 示例 |
|------|------|
| 需要用户选择 | 选项表格（场景、Skill、模型、评测集来源、是否一并打包依赖等） |
| 需要用户输入 | "请提供 Skill 名称"、"请提供自定义模型的模型 ID"、"请提供自定义模型的 API URL"、"请提供自定义模型的 API Key" |
| 需要用户确认 | "根据你的描述，你想要「快速测试单个 Skill」，是这样吗？"、任务摘要确认 |
| 进度表（仅阶段2/3/4） | 调用 TodoWrite 更新 4 阶段进度，无额外说明文字 |
| 流程入口（仅新建评测） | 调用 TodoWrite 更新 4 阶段进度，无额外说明文字 |

**所有其他内容不输出**：
- 推理过程、识别结论、执行状态、流程跳转、技术实现
- "已选 xxx"、"xxx 已就绪"等结果回显（除非用户明确要求查看）
- "正在 xxx"、"现在 xxx"、"接下来 xxx"等过程描述

**判断标准**：这句话是在问用户问题吗？不是 → 不输出。

### 渐进式交互

| 场景 | 处理方式 |
|------|----------|
| 可自动推断 | 直接处理，仅做结果确认 |
| 需用户确认 | 提供候选项供选择（编号选项表） |
| 需用户输入 | 直接请求文本输入 |

**判断标准**：能否提前列出所有有效答案？能 → 编号选项；不能 → 文本输入。

### 选项展示规范

**禁止使用 AskUserQuestion 工具**。所有交互通过文本输出 + 用户自然语言回复完成。

**所有需要用户选择的场景，统一使用编号表格**，例如：

> | 选项 | 内容 |
> |:----:|------|
> | 1 | xxx |
> | 2 | yyy |

不要出现"请输入 1 或 2"以外形式的指令。

### 自动填充规则（全局）

**每次进入新阶段或新步骤前，必须先检查上下文中是否已有该步骤所需的输入**。若已有，则自动填充并跳过对应询问。

| 步骤 | 可自动填充的上下文信息 |
|------|----------------------|
| 阶段 1：确认评测场景 | 用户输入中已识别出 Skill 名称/路径、模型名称、运行框架或评测意图 |
| 阶段 2 任务 1：选择 Skill | 用户输入中已提供 Skill 名称或路径 |
| 阶段 2 任务 2：选择运行框架 | 用户输入中已指定运行框架名称 |
| 阶段 2 任务 3：选择驱动模型 | 用户输入中已指定驱动模型名称 |
| 阶段 2 任务 4：选择评委模型 | 用户输入中已指定评委模型名称 |
| 阶段 3 任务 1：评测集来源 | 用户输入中已提供 Excel 文件路径或明确要求自动合成 |

自动填充时**不输出任何提示**，直接执行对应分支的后续操作。

### 静默执行原则

下列动作必须**全程静默**，不向用户输出"正在 xxx"等中间提示：

- 鉴权 Token 检查与刷新
- Skill 搜索、下载、依赖识别、打包、上传
- 运行框架配置构建
- 模型列表拉取、自定义模型保存
- 评测集 Excel 验证、转换、上传
- 评测任务提交与轮询
- 数据合成任务提交与轮询

仅在每个阶段任务完成时，向用户输出最终结果。

### 命令执行

**禁止猜测命令**：执行任何 `eval_skill.py` 子命令前，必须以文档中明确写出的命令为准。若当前文档未内联命令，必须先读取 [脚本定义.md](./references/脚本定义.md) 获取正确命令，**禁止凭记忆或推断构造命令**（如 `auth --check`、`token check`、`task list` 等不存在的子命令）。

**常用命令速查**（完整命令格式见 [脚本定义.md](./references/脚本定义.md)）：

| 功能 | 子命令 | 错误示例（禁止使用） |
|------|--------|----------------------|
| Token 检查 | `check-token` | ~~`auth --check`~~、~~`token check`~~ |
| 登录 | `login` | ~~`auth login`~~ |
| 查询历史任务 | `recent-tasks` | ~~`task list`~~、~~`tasks`~~ |
| 模型列表 | `models` | ~~`model list`~~ |
| 提交任务 | `submit` | ~~`task submit`~~ |
| 任务状态 | `status` | ~~`task status`~~ |
| 打包上传 | `package` | ~~`pack`~~、~~`upload`~~ |

**禁止复合命令**，逐条执行：

```bash
mkdir -p dir
{command-1}
{command-2}
```

### 可恢复性

失败时：清晰解释原因 → 说明当前进度 → 指明下一步操作 → 保留已完成产物。

## Red Flags

**通用违规行为**：

| 违规行为 | 简洁理由 |
|----------|----------|
| 向用户暴露脚本命令、JSON 字段、内部 ID | 实现细节属于内部状态，不应展示给用户 |
| 在用户可见输出中使用 `TokenPlan` / `EvalPlan` / `candidate` / `judge` 等开发态术语 | 这些是后端接口/字段名，用户无需也无法理解。对用户统一用"云端模型"、"自定义模型"、"驱动模型"、"评委模型" |
| 输出内部识别结论（如"xxx 无依赖，直接打包上传"） | 依赖识别、搜索结果判断等属于开发态信息，不应展示 |
| 输出执行流程的内部状态描述（如"登录成功！现在查询最近的评测任务"、"无历史任务，直接进入新建评测流程"、"现在读取场景识别文档"、"场景确认为 xxx，现在进入构建配置阶段"、"上下文已识别 Skill 名称为 xxx，直接进入搜索"） | 这些是执行过程的内部流转逻辑，不是用户需要的信息。用户只需看到选项表格、最终结果和关键确认点 |
| 输出推理过程或意图分析说明（如"用户描述是 xxx——单个 Skill，无多模型对比意图，可推断为快速效果验证场景"、"从输入提取出 Skill 名称为 xxx"） | 推理过程属于内部思考，用户只需看到最终的询问/确认句（如"根据你的描述，你想要「快速测试单个 Skill」，是这样吗？"）|
| 跳过自动填充检查直接询问 | 必须先检查上下文是否已有该步骤输入 |
| 输出"正在 xxx"等中间状态提示 | 静默执行原则 |
| 跳过场景确认直接进入配置 | 场景决定 Skill 与模型数量 |
| 跳过评委选择确认 | 评委选择影响评测客观性 |
| 驱动模型列表标注"（推荐）" | 驱动模型不得推荐，用户需自由选择；仅评委模型可推荐 |
| 驱动模型列表未按名称排序 | 必须按 `display_name` 字母顺序排序后展示 |
| TodoWrite 的 content 使用用户意图或自由文本（如"评测 testcase-skills 效果"） | content 必须固定为 4 个阶段名称：「确认评测场景」「配置评测对象」「准备评测数据」「提交并查看报告」 |

**常见借口**：

| 借口 | 现实 |
|------|------|
| "用户应该知道选哪个" | 用户可能不清楚概念，需展示选项 |
| "这个确认可以跳过" | 所有确认步骤都是必需的 |
| "默认配置就行" | 评委默认仅作为推荐，必须经用户确认 |
| "复合命令更快" | 复合命令难以调试，必须拆分 |

## 变量速查

> **Windows 路径注意**：构造 shell 命令时，所有包含路径的参数必须用双引号包裹，否则反斜杠会被当作转义。

| 变量 | 说明 | 示例 |
|------|------|------|
| `{skill-dir}` | 本技能目录的绝对路径 | `~/.claude/skills/skill-evaluation` |
| `{work-dir}` | 当前工作目录 | `/path/to/project` |
| `{python-cmd}` | Python 可执行文件 | `python` 或 `python3` |
| `{session-id}` | 本次评测会话 ID | `20260518_143022_xxx` |
| `{skill-name}` | 待评测 Skill 名称 | `xxx` |
| `{skill-zip-url}` | Skill zip 包 URL，多个时用逗号分隔 | `https://oss.example.com/skills/xxx.zip` |
| `{dataset-id}` | 评测集 ID（内部使用） | `eval-1776326400000000000` |
| `{task-id}` | 评测任务 ID（内部使用） | `eval-3g5k9m` |
