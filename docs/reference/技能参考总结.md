# 技能参考总结

本文档是对 `docs/reference/skills/` 下 16 个技能的系统性总结，用于快速定位可参考的实际案例。

---

## 一、按设计模式分类

### Generator（生成器）

接收输入 → 产出标准化产物，重心在产出质量控制。

| 技能 | 产物 | 关键约束 |
|------|------|----------|
| docx | .docx 文件 | XML unpack/repack 工作流、tracked changes 支持 |
| xlsx | .xlsx 文件 | 公式优先、recalc.py 强制校验、金融模型色码规范 |
| pptx | .pptx 文件 | Read→Edit→Create 分流、强制 QA subagent 审查 |
| pdf | .pdf 文件 | pypdf/pdfplumber/reportlab 分工、OCR 支持 |
| canvas-design | HTML Canvas 作品 | 设计哲学先行→技术表达、两轮 pass |
| algorithmic-art | p5.js 算法艺术 | 哲学驱动、seeded randomness 确保可复现 |
| slack-gif-creator | GIF 动画 | PIL GIFBuilder 封装、固定画布尺寸 |

**参考价值**：xlsx 的「公式优先 + 强制校验」模式类似 Pipeline 中的门控检查；pptx 的「强制 QA subagent」是产出质量保障的好范例。

### Tool Wrapper（工具包装）

封装复杂工具/API 为简洁接口，降低使用门槛。

| 技能 | 包装对象 | 特征 |
|------|----------|------|
| claude-api | Claude API 全接口 | 模型表、参数说明、语言检测、managed agents |
| mcp-builder | MCP Server 开发 | 4 阶段但非严格 Pipeline（Research→Implement→Review→Evaluate） |
| brand-guidelines | Anthropic 品牌资源 | 纯参考型，无脚本 |
| webapp-testing | Playwright 测试 | with_server.py 黑盒脚本、决策树引导 |

**参考价值**：webapp-testing 的「决策树 + 黑盒脚本」设计——用决策树减少文档篇幅，用脚本封装复杂操作。

### Inversion（反转）

先收集上下文/需求，再产出——「采访优先」模式。

| 技能 | 反转方式 | 特征 |
|------|----------|------|
| doc-coauthoring | 3 阶段：收集→精炼→读者测试 | 反复追问直到信息充分 |
| skill-creator | 循环迭代：draft→test→eval-viewer→feedback→improve | 持续反馈驱动改进 |

**参考价值**：skill-creator 的「GAN 式迭代」（生成→评估→改进循环）适用于需要持续优化的流程型技能。

### Hybrid / Router（混合/路由）

组合多种模式或根据输入分发到不同子流程。

| 技能 | 组合方式 | 特征 |
|------|----------|------|
| frontend-design | Wrapper + Generator：设计系统 + 构建产出 | 人格设定（Design Lead）、anti-AI-slop |
| web-artifacts-builder | Generator + Scaffold：init→develop→bundle→display | 固定技术栈（React+TS+Tailwind+Parcel） |
| theme-factory | Wrapper + Generator：预设主题库 + 自定义生成 | 展示→选择→应用三步 |
| internal-comms | Router + Generator：模板路由→填充生成 | 3P updates / newsletters / FAQs 三种模板 |

**参考价值**：frontend-design 展示了如何用人格设定统一多模式行为；internal-comms 展示了模板路由的简洁实现。

## 四、关键设计手法速查

| 手法 | 案例技能 | 描述 |
|------|----------|------|
| **决策树引导** | webapp-testing | 用 ASCII 树替代长篇条件描述 |
| **强制校验门控** | xlsx（recalc.py）| 产出必须通过校验脚本才算完成 |
| **QA Subagent** | pptx | 完成后自动调用 code-reviewer 检查 |
| **GAN 式迭代** | skill-creator | 生成→评估→反馈→改进循环 |
| **术语隔离** | （Pipeline 类通用） | 用户可见术语 vs 内部术语严格分离 |
| **静默执行** | （Pipeline 类通用） | 中间操作不产出用户可见文本 |
| **人格设定** | frontend-design | Design Lead 角色统一行为一致性 |
| **模板路由** | internal-comms | 根据输入类型选择预设模板 |
| **黑盒脚本** | webapp-testing | `--help` 即用，禁止阅读源码 |
| **anti-AI-slop** | frontend-design, web-artifacts-builder | 明确禁止紫色渐变、居中布局等 AI 特征 |
| **公式优先** | xlsx | 禁止 Python 硬算后写入，必须用 Excel 公式 |
| **两轮 Pass** | canvas-design | 第一轮生成、第二轮精炼 |

## 五、可借鉴的设计手法

1. **决策树替代长条件**（from webapp-testing）
   - 用 ASCII 决策树压缩复杂的条件分支描述

2. **黑盒脚本 + `--help` 优先**（from webapp-testing）
   - CLI 脚本作为黑盒调用，强化「先 --help 再执行」的规范

3. **强制校验门控**（from xlsx）
   - 每个阶段完成后增加校验步骤确保数据完整性

4. **GAN 式迭代**（from skill-creator）
   - 生成→评估→反馈→改进的循环驱动持续优化

### 潜在应用方向

| 方向 | 参考来源 | 预期收益 |
|------|----------|----------|
| 用决策树压缩条件分支 | webapp-testing | 减少文档长度，加快理解 |
| 阶段间增加校验脚本 | xlsx recalc.py | 提前发现数据错误 |
| 增加 QA subagent 审查 | pptx | 自动检测产出违规 |
| frontmatter description 详细化 | xlsx 的详细触发条件 | 提高触发精度 |
