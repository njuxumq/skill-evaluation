# 通用维度级评测配置流程

本文档定义通用维度级评测配置流程，根据评测场景构建覆盖全面的维度配置。

---

## 目标

选择维度 → 评估覆盖 → 补充缺失 → 设置权重 → 保存配置。

核心原则：**内置优先，按需补充，权重合理**。

---

## 触发条件

用户指明维度级评测，且未匹配到该评测方式下的专家模板。

---

## 红线

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| 未查阅内置维度直接配置 | 凭直觉选择维度 | 必须先查阅 `内置模板说明.md` |
| 维度覆盖不全 | 遗漏关键评测维度 | 必须与用户确认覆盖范围 |
| 权重未确认就保存 | 权重分配不合理 | 权重设置必须等待用户确认 ⚠️ |
| 未进行结构转化就保存 | 配置格式不符合 API 要求 | 必须按 `维度配置转化规则.md` 转化 |

---

## 步骤1：选择评测维度

**目的**：定位适用维度模板，了解内置维度结构。

根据用户场景，从内置评测维度模板中选择合适的维度：

1. 参考 [内置模板说明.md](../references/内置模板说明.md) 第2节"维度配置模板"
2. 根据评测场景匹配适用的内置维度

---

## 步骤2：评估维度覆盖情况

**目的**：判断内置维度覆盖程度，决定后续处理方式。

| 情况 | 说明 | 动作 |
|------|------|------|
| 全部内置 | 内置维度完全覆盖评测场景 | → 直接使用内置维度 |
| 部分覆盖 | 内置维度覆盖部分评测场景 | → 补充生成缺失维度 ⚠️ |
| 无匹配 | 无适用的内置维度 | → 全部自动生成 ⚠️ |

⚠️ **部分覆盖/无匹配时**：与用户确认需补充/生成的维度及评测类型。

**自动生成维度**：参考 [评测维度说明.md](../references/评测维度说明.md) 第1、2节。

---

## 步骤3：保存维度配置

**目的**：合并维度、设置权重并转化为 API 要求格式。

合并维度后设置权重（总和=1.0）。⚠️ **权重设置**：根据评测场景和维度特点提供默认权重方案，用户可调整各维度权重。

参考 [维度配置转化规则.md](../references/维度配置转化规则.md) 进行结构转化。

**保存路径**：`{work-dir}/.eval/{session-id}/eval-dimension.json`
