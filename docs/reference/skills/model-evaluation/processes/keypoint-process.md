# 评测点生成流程

根据问题、参考答案、上下文自动生成评测点，覆盖标准化评测集。

---

## 目标

场景判断 → 生成评测点 → 预览确认 → 保存结果。

核心原则：**必须调用模型生成，用户确认后方可保存**。

---

## 触发条件

定制用例级评测且标准化评测集无 keypoint 字段。

---

## 红线

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| 不调用模型直接编造评测点 | 评测点质量无保障 | 必须调用内置模型生成 |
| 跳过用户确认直接保存 | 用户无法审核质量 | 必须展示前3条并等待确认 |
| 跳过 SYSTEM_PROMPT 思考步骤 | 生成逻辑不完整 | 按提示词要求执行四步思考 |
| 凭直觉选择场景类型 | 字段判断错误 | 按判断逻辑表检测字段组合 |

---

## 步骤1：场景与去重判断

**目的**：识别字段组合确定场景类型，处理多模型横评去重。

| 状态 | 动作 |
|------|------|
| `case_id` 存在重复值 | → 按唯一 `case_id` 去重，生成后复制到同 `case_id` 所有记录 |
| `case_id` 无重复值 | → 直接进入字段组合判断 |

### 字段说明

| 字段 | 字母代表 | 说明 | 用途 |
|------|------|------|------|
| `question` | Q | 用户问题 | 评测的核心输入 |
| `answer` | - | 模型输出的回答 | 待评测内容，**不是参考答案** |
| `reference` | R | 参考答案/理想答案 | 用于提炼评测要点 |
| `context` | C | 上下文信息 | 用于理解问题背景 |

**重要区分**：
- `answer` 是待评测的模型输出，不属于场景判断的依据
- `reference` 才是参考答案，用于场景判断和评测点生成

### 场景类型

| 场景 | 包含字段 | 评估点来源 | 典型用途 |
|------|----------|------------|----------|
| Q | 仅 `question` | 基于问题推断 | 通用数据集（质量一般） |
| QR | `question` + `reference` | 从参考答案提炼关键信息点 | 通用数据集（推荐） |
| QC | `question` + `context` | 从上下文推导应提取的要点 | 私域数据集 |
| QRC | `question` + `reference` + `context` | 最佳效果 | 私域数据集（推荐） |

**判断逻辑**（忽略 `answer` 字段）：
```
if reference AND context: scene = "QRC"
elif reference: scene = "QR"
elif context: scene = "QC"
else: scene = "Q"
```

---

## 步骤2：生成评测点

**目的**：调用内置模型，基于场景类型生成 1-3 个评测点。

| 状态 | 动作 |
|------|------|
| 数据量 ≤ 50 条 | → 主 Agent 直接生成评测点 |
| 数据量 > 50 条 | → Agent 工具分批执行 |
| 主 Agent context 满载 | → 必须使用 SubAgent 分批生成 |
| 生成失败 | → 重试一次，仍失败则跳过并记录 |

### 执行方式

调用 `{skill-dir}/scripts/utils/keypoint_prompts.py`：

| 步骤 | 动作 |
|------|------|
| 准备提示词 | → 读取 `SYSTEM_PROMPT`，用 `build_user_prompt()` 构建 |
| 调用模型 | → Agent 工具（subagent_type=general-purpose）生成评测点 |

**提示词示例**：
```python
user_prompt = build_user_prompt(question, reference, context)
# Agent prompt: f"{SYSTEM_PROMPT}\n\n{user_prompt}"
```

### 输出格式

```json
["是否提及xxx", "是否包含xxx"]
```

---

## 步骤3：预览与确认

**目的**：展示评测点预览，等待用户确认质量。

| 状态 | 动作 |
|------|------|
| 用户确认 Y | → 进入步骤4保存结果 |
| 用户选择 n | → 取消生成，不修改文件 |
| 用户选择调整 | → 按需求重新生成评测点 |
| 用户选择查看更多 | → 展示更多数据评测点 |

⚠️ **必须展示前3条数据并等待用户确认，不可自动继续。**

展示前3条评测点预览：

| 序号 | 问题 | 生成的评测点 |
|------|------|--------------|
| 1 | 什么是机器学习？ | ["是否定义了机器学习", "是否提及与人工智能的关系"] |
| 2 | 如何评估模型性能？ | ["是否列举评估指标", "是否说明各指标适用场景"] |
| 3 | 深度学习有哪些应用？ | ["是否列举应用领域", "是否给出具体案例"] |

询问用户：
- Y：保存并覆盖原文件
- n：取消生成，不修改文件
- 调整：描述调整需求，重新生成
- 查看更多：展示更多数据的评测点

---

## 步骤4：保存结果

**目的**：将评测点写入标准化评测集文件。

| 状态 | 动作 |
|------|------|
| 多模型横评场景 | → 评测点复制到同 `case_id` 所有记录 |
| 单模型场景 | → 直接写入各记录 |

将 `keypoint` 字段添加到 `{work-dir}/.eval/{session-id}/evalset/evalset-standard.jsonl`：

**字段格式**：字符串数组转为 JSON 字符串存储。
- 示例：`["是否...", "是否..."]` 存储为 `"[\"是否...\", \"是否...\"]"`

**注意**：覆盖原文件，不创建新文件。

---

## 返回点

流程结束后返回评测集处理阶段的上传任务。

---

## 变量速查

| 变量 | 说明 |
|------|------|
| `{work-dir}` | 当前工作目录 |
| `{session-id}` | 会话目录名，格式 `session-{8位字母数字}` |
| `{skill-dir}` | 技能安装目录 |
