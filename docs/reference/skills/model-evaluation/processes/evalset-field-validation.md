# 字段校验流程

检查字段状态 → 处理缺失字段 → 确认评测模式 → 保存配置。

---

## 目标

校验字段状态 → 处理缺失字段 → 确认评测模式 → 保存配置。

核心原则：**字段状态决定处理分支，用户选择不可跳过**。

---

## 触发条件

评测集 answer 字段全部非空，需校验其他字段状态。

---

## 红线

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| model 字段为空但选择多模型横评 | 缺少模型标识无法进行横向对比 | 提示用户补充 model 字段后重新执行 |
| 未询问评测模式直接设置默认模型 | 用户未确认评测类型，可能导致评测方式错误 | 必须等待用户选择评测模式 |
| case_id 自动生成未告知用户 | 用户不知 case_id 来源，可能影响结果理解 | 明确告知自动生成规则 |

---

## 步骤1：检查 model 字段状态

**目的**：确定 model 字段是否需要补充或设置默认值。

| 状态 | 动作 |
|------|------|
| 不存在或全空 | → 进入步骤3（询问评测模式） |
| 已填充 | → 进入步骤2 |

---

## 步骤2：检查 case_id 字段状态

**目的**：确定 case_id 字段的处理方式。

| 状态 | 动作 |
|------|------|
| 已填充 | → 标准化时使用原值 |
| 不存在或全空 | → 标准化时按 question 分组自动生成 |

> 多模型横评时，同一问题的不同模型回答共享相同 case_id

---

## 步骤3：处理 model 字段为空 ⚠️

**目的**：确认评测模式，确定 model 字段的填充方式。

**条件**：步骤1判断 model 字段为空时执行。

询问评测模式：
> 请确认评测模式：
> 1. 单模型评测 - 评测单个模型的表现
> 2. 多模型横评 - 多个模型横向对比（需在评测集补充 model 字段）

**此步骤必须等待用户选择，不可跳过。**

| 选择 | 动作 |
|------|------|
| 1 | → 设置 `model.default` 为用户指定的模型名称 |
| 2 | → 提示用户补充 model 字段，重新执行步骤1 |

---

## 步骤4：更新字段映射配置

**目的**：将校验结果（如 model.default、case_id 处理方式）更新到字段映射配置中。

更新 `evalset-fields-mapping.json`，补充校验结果字段。

---

## 返回点

返回调用该流程处

| 变量 | 说明 |
|------|------|
| `{work-dir}` | 当前工作目录 |
| `{session-id}` | 会话目录名，格式 `session-{8位字母数字}` |
