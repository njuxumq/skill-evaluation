# 问题集生成流程

本文档定义生成问题集流程，AI 根据评测场景和评测维度从零生成仅包含问题的评测集。

---

## 触发条件

用户无评测集，需 AI 根据已确认的评测场景和评测维度生成问题集。

---

## 目标

生成包含 `question`、`case_id`、`category` 三字段的 JSONL 问题集，供后续评测使用。

**前置验证**：`eval-dimension.json` 存在。验证失败则提示用户先完成构建配置阶段（eval-build.md）。

---

---

## 步骤1：确认生成依据

**判断**：历史对话或从 `eval-dimension.json` 中提取评测场景和评测维度。

| 状态 | 判断 | 动作 |
|------|------|------|
| 场景+维度均已确认 | 文件存在且内容有效 | → 进入步骤2 |
| 场景未确认 | 无场景信息 | → 提示用户先完成场景确认 |
| 维度未确认 | 无维度信息 | → 提示用户先完成维度配置 |

**输出变量**：
- `{scene}`：已确认的评测场景描述
- `{dimensions}`：已确认的评测维度列表

---

## 步骤2：收集配置并生成

**目的**：一次性收集生成配置，确认后立即生成问题集。

**使用工具**：`AskUserQuestion`（多问题并行收集）

### 配置收集项

使用 AskUserQuestion 工具一次性收集以下配置：

| 项目 | 问题提示 | 默认值 | 是否必填 |
|------|----------|--------|----------|
| **数量** | 生成多少条问题？ | 10 | 是 |
| **类型分布** | 问题类型偏好？ | 均衡分布 | 否 |
| **难度分布** | 难度偏好？ | 均衡分布 | 否 |
| **其他选项** | 是否需要更多配置？ | 无 | 否 |

**类型分布选项**：均衡分布（默认）、偏重开放式、偏重封闭式、偏重场景化

**难度分布选项**：均衡分布（默认）、偏重简单、偏重中等、偏重困难

**其他选项展开**：

当用户选择"其他选项"时，展开以下配置：

| 子项 | 问题提示 | 默认值 |
|------|----------|--------|
| 分类标签 | 使用哪些 category 标签？ | 自动生成 |
| 示例问题 | 是否提供参考示例？ | 无 |
| 特殊约束 | 其他约束要求？ | 无 |

### 配置确认

收集配置后，展示配置摘要（数量、类型分布、难度分布、分类标签、示例问题、特殊约束），询问：

```
是否按此配置生成？(Y/n/调整)
```

- **Y** → 开始生成问题集
- **n** → 流程终止
- **调整** → 返回配置收集

### 生成问题集

用户确认后，AI 根据配置生成问题集。

**生成要求**：
- 问题覆盖评测维度关注点
- 问题风格与评测场景匹配
- 遵循用户提供的约束条件

**生成方式**：
- 数量 ≤50：主 Agent 直接生成
- 数量 >50：可选 SubAgent 分批生成

**字段格式**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `case_id` | `string` | 是 | 用例唯一标识，格式 `case-{序号}` |
| `question` | `string` | 是 | 问题内容 |
| `category` | `string` | 是 | 分类标签 |

---

## 步骤3：预览与保存

**目的**：让用户确认生成的问题集，确认后保存。

**⚠️ 不可跳过**：此步骤必须等待用户确认，不可直接保存。

### 预览内容

展示前 3 条问题和 category 分布统计：

```
生成完成，共 {数量} 条问题：

【预览前3条】
1. [知识问答] 请解释什么是机器学习？
2. [推荐建议] 为我推荐一个适合周末短途旅行的目的地。
3. [行程规划] 如何规划一个三口之家的日本七日游？

【分布统计】知识问答：3条、推荐建议：2条、行程规划：2条、创意生成：2条、其他：1条

是否确认保存？(Y/n/重新生成)
```

### 用户操作

| 操作 | 动作 |
|------|------|
| **Y（确认保存）** | → 保存文件，流程结束 |
| **n（取消）** | → 流程终止 |
| **重新生成** | → 返回步骤2调整配置 |

### 保存路径

```
{work-dir}/.eval/{session-id}/evalset/evalset-prepared.jsonl
```

## 产物定义

### 文件名

`evalset-prepared.jsonl`

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `case_id` | `string` | 是 | 用例唯一标识，格式 `case-{序号}` |
| `question` | `string` | 是 | 问题内容 |
| `category` | `string` | 是 | 分类标签 |

### 示例

```jsonl
{"case_id": "case-001", "question": "请解释什么是机器学习？", "category": "知识问答"}
{"case_id": "case-002", "question": "为我推荐一个适合周末短途旅行的目的地。", "category": "推荐建议"}
{"case_id": "case-003", "question": "如何规划一个三口之家的日本七日游？", "category": "行程规划"}
```

## 变量速查

| 变量 | 说明 |
|------|------|
| `{work-dir}` | 当前工作目录 |
| `{session-id}` | 会话目录名，格式 `session-{8位字母数字}` |
| `{scene}` | 已确认的评测场景描述 |
| `{dimensions}` | 已确认的评测维度列表 |

---

## 常见错误

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| 问题未覆盖评测维度 | 生成时未参考维度列表 | 检查维度映射，补充缺失维度的问题 |
| category 分布不均衡 | 未按用户配置分布生成 | 重新生成或调整配置 |
| 用户确认被跳过 | 直接保存未预览 | 步骤3标注 `⚠️ 不可跳过`，必须等待确认 |
| JSONL 格式无效 | 跨行 JSON 或字段缺失 | 确保每行独立完整 JSON 对象 |
| case_id 不唯一 | 重复序号 | 确保 case_id 按序递增，无重复 |
