# 内置模板说明

本文档说明评测系统中内置的专家模板和维度配置模板。

---

## 1. 专家模板

专家模板是针对特定评测场景预配置的完整评测方案，分为以下两类：

### 1.1 通用维度级评测

适合对场景化能力的宏观全面覆盖，如有效性、相关性、连贯性等通用评测维度。

| 模板名称 | 适用场景 | 模板位置 |
|----------|----------|----------|
| 内容创造 | 文章创作、文案生成、创意写作 | `assets/experts/内容创造.json` |
| 企业流程自动化 | 工具调用、API编排、业务流程自动化、Text2SQL | `assets/experts/企业流程自动化.json` |
| 内容匹配 | 标准问答、选择题、简答题（精确匹配） | `assets/experts/内容匹配.json` |
| 个性化规划与推荐 | 旅游规划、学习计划、购物推荐 | `assets/experts/个性化规划与推荐.json` |
| 信息聚合分析 | 数据分析、报告生成、竞品分析、市场研究 | `assets/experts/信息聚合分析.json` |
| 文本翻译 | 机器翻译、多语言内容转换 | `assets/experts/文本翻译.json` |
| 旅游出行 | 旅游攻略生成、行程规划、景点推荐 | `assets/experts/旅游出行.json` |
| 数学能力 | 数学题目解答、计算验证、答案正确性校验 | `assets/experts/数学能力.json` |
| 知识库问答 | 基于知识库的问答系统、检索增强生成评测 | `assets/experts/知识库问答.json` |
| 语言理解 | 指令遵循、内容约束、风格设定、格式合规评测 | `assets/experts/语言理解.json` |
| 逻辑推理 | 逻辑推理能力、推理过程正确性验证 | `assets/experts/逻辑推理.json` |

### 1.2 定制用例级评测

适合对场景化能力的细节偏好对齐，针对每条用例有定制的评测要点。

| 模板名称 | 适用场景 | 模板位置 |
|----------|----------|----------|
| 营销数字人评测 | 虚拟人多轮对话有效性评测 | `assets/experts/营销数字人评测.json` |
| 知识问答 | 通用知识问答、事实正确性检验 | `assets/experts/知识问答.json` |

**使用方式**：直接复制专家模板至会话目录，按需调整维度权重。

---

## 2. 维度配置模板

维度配置模板是可复用的单一评测维度定义，可灵活组合。分为以下两类：

### 2.1 通用维度级评测模板

适合对场景化能力的宏观全面覆盖，不依赖评测要点，可直接应用于整个评测集。

#### 分类概览

| 分类名称 | 分类说明 | 典型维度 | 适用场景示例 |
|----------|----------|----------|--------------|
| 质量评估类 | 评估内容本身的质量特征，如相关性、逻辑性、完整性、创新性等 | 相关性、逻辑连贯性、完整性、创新性 | 内容创造、信息聚合分析、旅游出行 |
| 效果验证类 | 验证是否达到预期目标，如解决问题、信息准确、答案正确等 | 有效性、准确性、正确性 | 企业流程自动化、知识库问答、数学能力 |
| 约束遵循类 | 检查是否遵循规则或约束，如指令要求、格式规范、内容限制等 | 指令遵循、内容约束、格式合规性 | 语言理解、企业流程自动化、内容匹配 |
| 语义相似度类 | 计算文本相似程度，基于内置函数进行客观度量 | BLEU、ROUGE、BERTScore、COMET | 文本翻译 |

评估内容本身的质量特征，如相关性、逻辑性、完整性、创新性等。

|----------|----------|----------|--------------|

| 相关性 | 推荐内容是否与用户需求强相关，是否紧扣用户问题回答，避免推荐无关或不合适的内容 | 个性化规划与推荐 | `assets/dimensions/相关性-个性化规划与推荐.json` |
| 相关性 | 基于知识库回答时，助手响应是否和用户问题强相关，是否紧扣用户问题回答 | 知识库问答 | `assets/dimensions/相关性-知识库问答.json` |
| 逻辑连贯性 | 助手响应内容组织合理、推理逻辑合理、内容连贯且上下文衔接流畅 | 内容创造 | `assets/dimensions/逻辑连贯性-内容创造.json` |
| 逻辑连贯性 | 多步骤流程编排的合理性，工具调用顺序是否符合逻辑，是否存在前后矛盾或循环依赖 | 企业流程自动化 | `assets/dimensions/逻辑连贯性-企业流程自动化.json` |

| 逻辑合理性 | 规划的行程需符合逻辑合理性，包括逻辑与行程合理性，如时间合理，路线可执行，行程整体闭环 | 旅游出行 | `assets/dimensions/逻辑合理性-旅游出行.json` |
| 完整性 | 大模型回复是否完整覆盖用户问题意图，包括用户明确提及的所有需求点 | 个性化规划与推荐 | `assets/dimensions/完整性-个性化规划与推荐.json` |
| 完整性 | 是否覆盖用户问题的所有维度，是否遗漏关键分析点 | 信息聚合分析 | `assets/dimensions/完整性-信息聚合分析.json` |
| 创意性/吸引性 | 结合对话场景评估助手的回答是否有吸引力、趣味性或新奇度，语言表达与创意能否抓住用户 | 内容创造 | `assets/dimensions/创意性-吸引性-内容创造.json` |

| 核心元素 | 助手响应内容中核心元素的覆盖情况，包含交通、景点信息、行程规划、住宿安排、饮食推荐 | 旅游出行 | `assets/dimensions/核心元素-旅游出行.json` |
| 忠诚度 | 回复结果涉及到搜索带有溯源的标识的内容进行检测（如引用链接、来源标识等） | 旅游出行 | `assets/dimensions/忠诚度-旅游出行.json` |
| 内容相关性 | 助手响应内容和用户问题的相关性，如时间、路线、途经点匹配度 | 旅游出行 | `assets/dimensions/内容相关性-旅游出行.json` |
| 形式相关性 | 助手响应内容的形式是否满足要求，是否为旅游行程攻略 | 旅游出行 | `assets/dimensions/形式相关性-旅游出行.json` |

## 3. 维度类型说明

### 3.1 评测类型（type字段）

| 类型 | 说明 |
|------|------|
| 通用维度级评测 | 宏观全面覆盖，不依赖评测要点，可直接应用于整个评测集 |
| 定制用例级评测 | 细节偏好对齐，依赖评测要点，需针对每条用例配置评测要点 |

### 3.2 评测方式（config.type字段）

| 类型 | 说明 | 输出形式 |
|------|------|----------|
| llm-score | LLM评委打分 | 1-5分 |
| llm-judge | LLM评委判断 | 通过/不通过 |
| agent-score | Agent评委打分（需外部数据源） | 1-5分 |
| builtin | 内置函数计算 | 数值指标 |

---

## 4. 使用注意事项

- 同名维度在不同场景下有不同的定义和评分标准，使用时请注意区分
- Agent类型维度需配置外部数据源
- Builtin类型维度依赖评测服务支持对应函数
- 自定义评测方案时，维度权重总和应为1.0
