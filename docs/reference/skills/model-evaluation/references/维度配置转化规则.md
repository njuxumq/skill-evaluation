# 维度配置结构转化规则

维度模板文件与会话目录下的 `eval-dimension.json` 结构不同，保存前需转化。

---

## 转化步骤

1. 提取模板的 `config` 字段内容作为维度对象
2. 将维度对象放入 `evals` 数组
3. 从 `eval-judge.json` 读取 `judge_id` 并填充

---

## 结构对比

| 对比项 | 模板文件 | 目标文件 |
|--------|----------|----------|
| 根节点 | `config: {...}` | `evals: [{...}]` |
| judge_id | 空字符串 | 从 eval-judge.json 读取 |

---

## 单维度转化示例（定制用例级）

模板文件：
```json
{
    "name": "有效性（用例级）",
    "description": "...",
    "config": {
        "name": "有效性",
        "type": "llm-judge",
        "judge_id": "",
        ...
    }
}
```

目标文件：
```json
{
    "name": "有效性（用例级）",
    "description": "...",
    "evals": [
        {
            "name": "有效性",
            "type": "llm-judge",
            "judge_id": "ID_AJ001",
            ...
        }
    ]
}
```

---

## 多维度合并示例（通用维度级）

多个模板文件的 `config` 内容合并到 `evals` 数组：
```json
{
    "name": "知识问答",
    "description": "...",
    "evals": [
        { "name": "准确性", "type": "llm-score", "judge_id": "", ... },
        { "name": "完整性", "type": "llm-score", "judge_id": "", ... },
        { "name": "格式规范", "type": "builtin", "func": "JSONFORMAT", ... }
    ]
}
```

---

## 保存前检查

- [ ] `evals` 为数组结构（非 `config` 嵌套）
- [ ] `judge_id` 已填充（非空字符串，客观评测除外）
- [ ] 多维度时权重总和为 1.0
