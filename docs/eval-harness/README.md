# Skill 迭代验证体系

## 目标

使 skill-evaluation 技能的迭代优化 **可验证、可控、可评判**，而非凭感觉优化。

## 核心思路

Skill 的「被测对象」是 Claude 在特定文档约束下的运行时行为。通过驱动真实 Agent CLI（Claude Code / OpenClaw / Hermes）执行 Skill，收集完整 Transcript，用规则引擎自动验证 Claude 是否遵守 Skill 文档的约束。

**执行引擎**：复用 [astroneval](../../) 的 Agent 框架，用 `claude -p` 驱动真实 CLI，用 mock 脚本替换 `eval_skill.py` 来控制环境状态。

## 体系概览

```
┌────────────────────────────────────────────────────────┐
│  三层验证体系                                           │
├────────────────────────────────────────────────────────┤
│                                                        │
│  Layer 1: 静态一致性校验（自动化，每次修改必跑）         │
│     → 文档交叉引用、术语合规、结构完整                   │
│                                                        │
│  Layer 2: 真实 CLI 驱动 + Transcript 验证（半自动化）   │
│     → 驱动 Agent CLI 执行 Skill，收集 transcript       │
│     → 规则引擎对 transcript 判定 PASS/FAIL             │
│                                                        │
│  Layer 3: 版本对比评估（LLM-as-Judge）                 │
│     → 修改前后 A/B 打分，量化改进                       │
│                                                        │
└────────────────────────────────────────────────────────┘
```

## 快速开始

```bash
# 1. 静态校验（秒级，零成本）
python eval/lint_skill.py

# 2. 执行行为用例（分钟级，API 费用 < ¥2）
python eval/run.py --version v2.1

# 3. 版本对比（自动生成回归报告）
python eval/compare.py --baseline v2.0 --candidate v2.1
```

## 合并标准

| 条件 | 要求 |
|------|------|
| 静态校验 | 零 error |
| P0 用例 | 全部 PASS（3/3 通过） |
| 回归检测 | 无回归 |
| 软分对比 | 不低于 baseline |
