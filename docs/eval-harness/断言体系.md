# 断言体系

## 概述

断言分为两类：

| 类型 | 判定方式 | 结果 | 用途 |
|------|----------|------|------|
| **硬约束** | 规则引擎（正则、结构检查） | PASS / FAIL | 绝对不可违反的约束 |
| **软约束** | LLM-as-Judge 打分 | 1-5 分 | 质量维度评估 |

---

## 硬约束（规则引擎）

### 作用域说明

所有硬约束在 `ParsedTranscript.skill_scope` 内执行——即 Skill 工具被调用**之后**的事件序列。Skill 加载前的 Read/Glob 等准备动作不在断言范围内。

### 规则清单

以下规则直接从 SKILL.md 的「核心不变量」和「Red Flags」提取：

#### H0: Skill 必须被加载（前置检查）

```python
def check_skill_loaded(transcript):
    """Phase 0 前置检查：Skill 是否被触发加载。
    
    此规则失败时用例裁决为 SKIP（非 FAIL），
    表明 Skill 的 description 字段对该输入匹配力不足。
    """
    assert transcript.skill_loaded, "Skill 未被触发加载"
```

**来源**：框架前置条件（Skill 未加载则后续断言无意义）

**裁决语义**：H0 失败 → 用例裁决为 **SKIP**（非 FAIL）。触发率作为独立质量指标统计。

#### H1: Skill 作用域内首动作必须是 TodoWrite

```python
def check_first_action_is_todowrite(transcript):
    """检查 Skill 加载后的第一个工具调用是否为 TodoWrite。
    
    注意：此断言在 skill_scope 内执行，自动排除了
    Skill 加载前的 Read/Glob 等文档加载动作。
    """
    actions = transcript.actions  # 已限定在 skill_scope 内
    assert len(actions) > 0, "Skill 作用域内无任何工具调用"
    assert actions[0].tool == "TodoWrite", (
        f"Skill 作用域内首动作应为 TodoWrite，实际为 {actions[0].tool}"
    )
```

**来源**：SKILL.md「加载即静默 — 第一件事是调用 TodoWrite」

#### H2: TodoWrite 前无文本输出

```python
def check_no_text_before_todowrite(transcript):
    """检查 Skill 作用域内，TodoWrite 调用之前是否有文本输出。"""
    text_before = transcript.text_before_tool("TodoWrite")
    assert text_before.strip() == "", (
        f"Skill 作用域内 TodoWrite 前有文本输出: {text_before[:80]}"
    )
```

**来源**：SKILL.md「在调用 TodoWrite 之前，禁止输出任何文本」

#### H3: TodoWrite content 固定

```python
FIXED_CONTENTS = ["确认评测场景", "配置评测对象", "准备评测数据", "提交并查看报告"]

def check_todowrite_content(transcript):
    call = transcript.first_tool_call("TodoWrite")
    contents = [item["content"] for item in call.params["todos"]]
    assert contents == FIXED_CONTENTS
```

**来源**：SKILL.md「交互约束 - TodoWrite content 固定为 4 个阶段名称」

#### H4: 禁用术语不出现

```python
FORBIDDEN_TERMS = [
    "TokenPlan", "EvalPlan", "limited_free",
    "candidate", "judge", "usage", "source",
    "recommend_model_id", "dataset-id", "task-id",
]

def check_no_forbidden_terms(text):
    for term in FORBIDDEN_TERMS:
        assert term not in text, f"术语泄露: {term}"
```

**来源**：SKILL.md「术语约束」

#### H5: 禁止过程泄露

```python
FORBIDDEN_PATTERNS = [
    r"正在(检查|加载|读取|上传|执行|搜索|打包)",
    r"现在(进入|读取|检查|执行)",
    r"接下来(我|需要|进入|执行)",
    r"(已|已经)(识别|确认|检测|加载)",
    r"(从|根据).{2,20}(提取|识别|推断)出",
    r"(进入|跳转到|跳过).{0,10}(阶段|流程|步骤|分支)",
    r"(用户|你)(说的是|描述的是|的意图是|想要的是)",
    r"Token\s*(检查|有效|通过|失效)",
    r"鉴权(完成|通过|成功)",
    r"(无|没有)(历史|最近)任务",
]

def check_no_process_disclosure(text):
    for pattern in FORBIDDEN_PATTERNS:
        match = re.search(pattern, text)
        assert not match, f"过程泄露: {match.group()}"
```

**来源**：SKILL.md「输出约束 - 禁止输出思考过程」+ Red Flags

#### H6: 禁用 AskUserQuestion

```python
def check_no_ask_user_question(transcript):
    assert not transcript.has_tool_call("AskUserQuestion")
```

**来源**：SKILL.md「工具使用限制」

#### H7: 选项必须用 Markdown 表格

```python
def check_options_use_table(text):
    numbered_list = re.findall(r"^\d+[\.\)]\s+\S+", text, re.MULTILINE)
    if numbered_list and "| 选项 |" not in text and "|:----:|" not in text:
        assert False, f"选项使用了编号列表而非 Markdown 表格"
```

**来源**：SKILL.md「首个输出格式硬性要求」

#### H8: 驱动模型列表不标注推荐

```python
def check_no_recommend_in_driver_models(text):
    if "驱动模型" in text or "选择模型" in text:
        assert "推荐" not in text and "（推荐）" not in text
```

**来源**：SKILL.md Red Flags「驱动模型列表标注推荐」

#### H9: 自然语言引导使用占位符

```python
def check_guide_uses_placeholder(text):
    if "请描述你的评测需求" in text:
        assert "xxx Skill" in text, "自然语言引导未使用 xxx Skill 占位符"
```

**来源**：processes/scene-detection.md「步骤1 - 禁止替换为具体名称」

#### 分支正确性验证（设计模式说明）

分支正确性不通过中心化规则验证，而是由每个用例的 `custom` 断言直接表达。原因：

1. **条件由 fixture 隐含**：`recent-tasks` 返回空列表 = 无历史；返回有内容 = 有历史。用例编写者已知正确分支
2. **每个分支的"正确输出"因场景而异**：无法用一条通用规则覆盖所有分支组合
3. **`custom` 断言已精确表达期望**：`text_present` / `text_absent` 直接验证分支输出

因此用例中不声明 `H10`，而是通过 `custom` 断言组合表达分支正确性。示例见 [用例规范.md](./用例规范.md) 中的 init_02（有历史）和 init_04（模糊输入）。

### 规则运行器

```python
class HardConstraintValidator:
    """两阶段规则验证器。

    Phase 0: 检查 Skill 是否加载（H0）
      - 失败 → 返回 SKIP（非 FAIL）
    Phase 1: 在 skill_scope 内执行行为规则（H1-H10）
      - 通用规则（H1-H6）：所有用例自动执行
      - 条件规则（H7-H10）：按用例 assertions.hard.rules 指定
    """

    # Phase 0: 前置检查
    PHASE_0 = check_skill_loaded                        # H0

    # Phase 1: 通用规则（所有用例默认执行，无需在 YAML 中声明）
    UNIVERSAL_RULES = [
        check_first_action_is_todowrite,                # H1
        check_no_text_before_todowrite,                 # H2
        check_todowrite_content,                        # H3
        check_no_forbidden_terms,                       # H4
        check_no_process_disclosure,                    # H5
        check_no_ask_user_question,                     # H6
    ]

    # Phase 1: 条件规则（按用例指定执行）
    CONDITIONAL_RULES = {
        "H7": check_options_use_table,
        "H8": check_no_recommend_in_driver_models,
        "H9": check_guide_uses_placeholder,
    }

    def check(self, transcript: ParsedTranscript, case: TestCase) -> HardResult:
        # Phase 0: Skill 加载检查
        try:
            self.PHASE_0(transcript)
        except AssertionError as e:
            return HardResult.skip(str(e))

        # Phase 1: 行为规则（在 skill_scope 内执行）
        results = []

        # 通用规则（自动执行）
        for rule in self.UNIVERSAL_RULES:
            results.append(self._run_rule(rule, transcript, case))

        # 条件规则（用例指定）
        for rule_id in case.assertions.hard.get("rules", []):
            if rule_id in self.CONDITIONAL_RULES:
                rule = self.CONDITIONAL_RULES[rule_id]
                results.append(self._run_rule(rule, transcript, case))

        # 自定义断言
        for custom in case.assertions.hard.get("custom", []):
            results.append(self._run_custom(custom, transcript))

        return HardResult(results)

    def _run_rule(self, rule, transcript, case) -> RuleResult:
        try:
            rule(transcript, case)
            return RuleResult(rule.__name__, PASS)
        except AssertionError as e:
            return RuleResult(rule.__name__, FAIL, str(e))

    def _run_custom(self, assertion: dict, transcript) -> RuleResult:
        """执行用例 YAML 中定义的自定义断言"""
        # 实现 text_present / text_absent / regex_match / tool_called 等
        ...
```

## 软约束（LLM-as-Judge）

### 评分维度

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| **简洁度** | 30% | 输出是否精练？有无多余的解释、过渡句？ |
| **流程正确性** | 30% | 是否按正确的条件分支执行？ |
| **交互自然度** | 20% | 中文表达自然？格式规范？ |
| **用户体验** | 20% | 用户能一步到位理解和选择？ |

### 评分标准

#### 简洁度 (conciseness)

| 分数 | 标准 |
|------|------|
| 5 | 只有选项表/提问/结果，零冗余 |
| 4 | 有极少量过渡（1 句），不影响体验 |
| 3 | 有少量过渡句但不影响理解 |
| 2 | 有明显冗余内容（状态说明、流程描述） |
| 1 | 大段解释、流程说明、状态回显 |

#### 流程正确性 (correctness)

| 分数 | 标准 |
|------|------|
| 5 | 完全符合期望的分支路径和步骤顺序 |
| 4 | 路径正确，有一处极小偏差 |
| 3 | 路径正确但有一处明显偏差 |
| 2 | 走错分支但最终回到正确路径 |
| 1 | 走错分支或跳过必要步骤 |

#### 交互自然度 (naturalness)

| 分数 | 标准 |
|------|------|
| 5 | 措辞自然、表格格式正确、全中文 |
| 4 | 措辞自然，有极小格式瑕疵 |
| 3 | 措辞略生硬或有小格式问题 |
| 2 | 明显不自然或中英混排 |
| 1 | 表达混乱或格式严重错误 |

#### 用户体验 (ux)

| 分数 | 标准 |
|------|------|
| 5 | 用户能一步到位理解和选择 |
| 4 | 用户稍加阅读即可理解 |
| 3 | 需要稍加思考但能完成 |
| 2 | 用户可能需要追问才能理解 |
| 1 | 用户会困惑或被误导 |

### Judge Prompt

```text
你是 Skill 行为质量评审员。请评估以下 Claude 与用户的交互记录。

## 评分维度（每项 1-5 分）

1. **简洁度**：输出是否精练？有无多余的解释、过渡句、状态描述？
2. **流程正确性**：是否按正确的条件分支执行？
3. **交互自然度**：中文表达是否自然？格式是否规范？
4. **用户体验**：从用户视角，交互是否高效有帮助？

## 测试用例背景
{case_description}

## 期望行为
{expected_behaviors}

## 实际交互记录
{transcript}

逐维度评分，输出 JSON：
{"conciseness": X, "correctness": X, "naturalness": X, "ux": X, "issues": [...]}
```

### 评分聚合

```python
WEIGHTS = {
    "conciseness": 0.30,
    "correctness": 0.30,
    "naturalness": 0.20,
    "ux": 0.20,
}

def weighted_score(scores: dict) -> float:
    return sum(scores[dim] * w for dim, w in WEIGHTS.items())
```

## 在用例中引用断言

```yaml
# 用例 YAML 中的断言配置
assertions:
  hard:
    # rules 只声明条件规则（H7-H9）
    # H0（Skill 加载）由 Phase 0 自动执行，失败 → SKIP
    # H1-H6（通用规则）由 Phase 1 自动执行，无需声明
    rules: ["H7", "H9"]

    # 用例特定断言（表达分支正确性和场景期望）
    custom:
      - type: text_present
        pattern: "最近的评测任务"
        scope: first_text_output
      - type: text_absent
        pattern: "请描述你的评测需求"
        scope: all_text

  # 软约束评分（默认全部 4 维度）
  soft: all
```
